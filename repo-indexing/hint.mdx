# Hint Component

## Overview

### Purpose
Use this component to generate hint functionality

### When to Use
The Hint component is essential for providing contextual guidance and additional information to users when filling out forms. It displays helpful text underneath form editors to improve user experience and reduce errors. Common use cases include:

- **Form field guidance** - Provide instructions or examples for complex input fields like passwords, emails, or formatted data
- **Validation hints** - Show format requirements before users submit forms (e.g., "Password must be at least 8 characters")
- **Contextual help** - Offer additional information about specific form fields without cluttering the interface
- **Input examples** - Display sample formats for dates, phone numbers, or other structured data
- **Character limits** - Show remaining characters or maximum length information
- **Accessibility support** - Provide screen reader accessible descriptions using aria-describedby relationships
- **Progressive disclosure** - Reveal helpful information when users focus on specific fields
- **Error prevention** - Guide users toward correct input formats before validation errors occur
- **Onboarding assistance** - Help new users understand what information is expected in each field

### Architecture
**No Coupling**: Hint is a standalone component that operates independently without requiring other components. It provides accessible hint text functionality that can be associated with any form editor through proper ARIA attributes.

## Components

### Hint

**Purpose:** Displays hint text underneath form editors to provide users with helpful guidance, instructions, or contextual information about the expected input.

**Interface:**
```typescript
interface HintProps {
  // Accessibility and association
  id?: string; // Unique identifier - should be set to editor's aria-describedby property
  
  // Content and behavior
  children: React.ReactNode; // The hint text or content to display
  
  // Visual customization
  direction?: 'start' | 'end'; // Text alignment (default: 'start')
  className?: string; // Custom CSS classes for styling
  style?: React.CSSProperties; // Inline styles for positioning and appearance
  
  // State integration
  editorDisabled?: boolean; // Reflects disabled state of associated editor
}
```

**Usage Example:**
```tsx
import { Hint } from '@progress/kendo-react-labels';
import { TextBox } from '@progress/kendo-react-inputs';

function BasicHintExample() {
  const hintId = 'password-hint';
  
  return (
    <div>
      <TextBox 
        placeholder="Enter password"
        aria-describedby={hintId}
      />
      <Hint id={hintId}>
        Password must be at least 8 characters with one uppercase letter
      </Hint>
    </div>
  );
}
```

### Dynamic Hint Content
```tsx
import { Hint } from '@progress/kendo-react-labels';
import { TextBox } from '@progress/kendo-react-inputs';
import { useState } from 'react';

function DynamicHintExample() {
  const [value, setValue] = useState('');
  const maxLength = 50;
  const remaining = maxLength - value.length;
  
  return (
    <div className="form-field">
      <label>Description</label>
      <TextBox 
        value={value}
        onChange={(e) => setValue(e.target.value)}
        aria-describedby="description-hint"
        maxLength={maxLength}
      />
      <Hint id="description-hint">
        {remaining > 0 
          ? `${remaining} characters remaining`
          : 'Maximum length reached'
        }
      </Hint>
    </div>
  );
}
```

```tsx
function DisabledStateExample() {

        >

      </div>

  );
```


## Best Practices

### Accessibility Guidelines
- **Always use id and aria-describedby** - Connect hints to their editors using proper ARIA relationships for screen reader compatibility
- **Provide meaningful content** - Write clear, concise hints that actually help users understand what's expected
- **Multiple hints support** - When using multiple hints for one field, include all hint IDs in the editor's aria-describedby attribute
- **Reflect editor state** - Use the editorDisabled prop to ensure hints visually reflect their associated editor's state

### Content Guidelines
- **Be concise and specific** - Provide just enough information to guide users without overwhelming them
- **Use positive language** - Frame hints as helpful guidance rather than warnings when possible
- **Include examples** - Show format examples for complex input requirements like dates or phone numbers
- **Progressive information** - Consider showing basic hints initially and more detailed ones on focus

### Integration Guidelines
- **Consistent positioning** - Place hints consistently below their associated editors throughout your application
- **Visual hierarchy** - Style hints to be clearly secondary to labels but still readable
- **Responsive design** - Ensure hints remain legible and properly positioned across different screen sizes
- **State synchronization** - Keep hint content and styling synchronized with editor states (disabled, invalid, etc.)

### Character Counter Hint
```tsx
// Dynamic character counting with live updates
function CharacterCounterHint({ value, maxLength }) {
  const remaining = maxLength - value.length;
  
  return (
    <Hint id="description-counter">
      {remaining >= 0 
        ? `${remaining} characters remaining`
        : `${Math.abs(remaining)} characters over limit`
      }
    </Hint>
  );
}
```
