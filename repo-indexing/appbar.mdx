# AppBar Components

## Overview

### Purpose
Use this component group to generate app bar functionality with sections and spacers

### When to Use
The AppBar component group is ideal for creating application headers that display:
- Branding and title information
- Primary navigation actions (menu toggles, search)
- User actions and notifications (profiles, alerts, settings)
- Application status and breadcrumbs
- Any combination of controls that need to be consistently positioned at the top or bottom of the screen

Perfect for modern web applications that need a responsive, accessible header bar with flexible content arrangement and sticky positioning capabilities.

### Architecture
**Critical/Strong Coupling**: The AppBar component system follows a container-children pattern where AppBar serves as the root container, and AppBarSection and AppBarSpacer work together to create flexible, responsive layouts.

### Component Interdependencies
- **AppBar** serves as the root container providing layout context and positioning
- **AppBarSection** groups related content within logical sections
- **AppBarSpacer** provides flexible spacing and pushes sections apart
- All components work together to create responsive, accessible app bars with proper semantic structure

## Components

### AppBar {Primary Component}

**Purpose:** Root container that provides the main app bar layout, positioning, and theming context

**Interface:**
```typescript
interface AppBarProps {
  children?: any; // AppBarSection and AppBarSpacer components
  className?: string; // Additional CSS classes
  style?: React.CSSProperties; // Additional CSS styles
  id?: string; // Root element ID
  position?: 'top' | 'bottom'; // Positioning on screen (default: 'top')
  positionMode?: 'static' | 'sticky' | 'fixed'; // CSS positioning behavior (default: 'static')
  themeColor?: 'base' | 'primary' | 'secondary' | 'tertiary' | 'info' | 'success' | 'warning' | 'error' | 'dark' | 'light' | 'inherit' | 'inverse'; // Theme color (default: 'light')
}
```

**Dependencies:** None - serves as the root component
**Provides:** Layout context and positioning for child sections and spacers

**Usage Example:**
```tsx
import { AppBar } from '@progress/kendo-react-layout';

function Example() {
  return (
    <AppBar
      position="top"
      positionMode="sticky"
      themeColor="primary"
    >
      {/* AppBarSection and AppBarSpacer components */}
    </AppBar>
  );
}
```

### AppBarSection {Dependent Component}

**Purpose:** Groups related content within logical sections of the app bar

**Interface:**
```typescript
interface AppBarSectionProps {
  children?: any; // Content to display in this section
  className?: string; // Additional CSS classes
  style?: React.CSSProperties; // Additional CSS styles
}
```

**Context Dependencies:** Must be used within AppBar for proper styling and layout
**Parent Requirements:** AppBar component for proper functionality

**Usage Example:**
```tsx
// Must be used within AppBar
<AppBar>
  <AppBarSection>
    <span className="k-icon k-i-menu" />
  </AppBarSection>
</AppBar>
```

### AppBarSpacer {Dependent Component}

**Purpose:** Provides flexible spacing between AppBar sections and pushes content apart

**Interface:**
```typescript
interface AppBarSpacerProps {
  className?: string; // Additional CSS classes
  style?: React.CSSProperties; // Additional CSS styles for width customization
  children?: React.ReactNode; // Optional content within the spacer
}
```

**Context Dependencies:** Must be used within AppBar for proper layout behavior
**Parent Requirements:** AppBar component for proper spacing functionality

**Usage Example:**
```tsx
// Must be used within AppBar
<AppBar>
  <AppBarSection>Left content</AppBarSection>
  <AppBarSpacer style={{ width: 16 }} />
  <AppBarSection>Right content</AppBarSection>
</AppBar>
```

## Component Relationships

### Data Flow
- AppBar provides the overall layout container and positioning context
- AppBarSection components contain discrete groups of related content
- AppBarSpacer components create flexible spacing and push sections apart
- No data flows between components - each maintains its own content

### Context Sharing
- AppBar provides CSS classes and positioning context inherited by child components
- Theme colors from AppBar affect the overall appearance of the bar
- All child components inherit responsive behavior from the parent AppBar

### Event Handling
- Events bubble up from section content to the AppBar container
- Individual sections handle their own content events independently
- AppBar positioning modes affect scroll and resize event handling

## Best Practices

### Usage Guidelines
- Always use AppBarSection to group related content for semantic structure and accessibility
- Utilize AppBarSpacer to create flexible layouts that adapt to different screen sizes
- Place primary branding/navigation on the left, secondary actions on the right
- Use sticky or fixed positioning for app bars that should remain visible during scroll

### Accessibility
- AppBar automatically provides proper landmark navigation structure
- Use semantic HTML within AppBarSection components (nav, h1, button elements)
- Ensure sufficient color contrast when using theme colors
- Include proper ARIA labels for icon-only buttons within sections

### Performance
- Minimize content within AppBar to reduce layout reflow during scroll events
- Use AppBarSpacer instead of CSS margins/padding for responsive spacing
- Consider positionMode impact on performance - fixed positioning may cause repaints
- Keep AppBar content lightweight for optimal sticky/fixed positioning performance

## Common Patterns

### Pattern 1: Responsive Navigation
```tsx
// Mobile-first approach with collapsible navigation
function ResponsiveNavigation() {
  const [isMenuOpen, setIsMenuOpen] = React.useState(false);
  
  return (
    <AppBar position="top" positionMode="sticky" themeColor="primary">
      <AppBarSection>
        <button 
          className="k-button k-button-flat"
          onClick={() => setIsMenuOpen(!isMenuOpen)}
        >
          <span className="k-icon k-i-menu" />
        </button>
      </AppBarSection>
      
      <AppBarSection>
        <h1>App Name</h1>
      </AppBarSection>
      
      <AppBarSpacer />
      
      <AppBarSection className="desktop-nav">
        <nav>
          <a href="/dashboard">Dashboard</a>
          <a href="/reports">Reports</a>
          <a href="/settings">Settings</a>
        </nav>
      </AppBarSection>
    </AppBar>
  );
}
```

### Pattern 3: Context-Aware Header
```tsx
// Header that changes based on current page/context
function ContextAwareHeader({ currentPage, breadcrumbs }) {
  return (
    <AppBar position="top" positionMode="sticky">
      <AppBarSection>
        <nav aria-label="Breadcrumb">
          {breadcrumbs.map((crumb, index) => (
            <span key={index}>
              <a href={crumb.url}>{crumb.title}</a>
              {index < breadcrumbs.length - 1 && ' > '}
            </span>
          ))}
        </nav>
      </AppBarSection>
      
      <AppBarSpacer />
      
      <AppBarSection>
        {currentPage === 'edit' && (
          <>
            <button className="k-button">Save Draft</button>
            <button className="k-button k-primary">Publish</button>
          </>
        )}
        {currentPage === 'view' && (
          <button className="k-button">Edit</button>
        )}
      </AppBarSection>
    </AppBar>
  );
}
```
