# Reveal Component

## Overview

### Purpose
Use this component to generate reveal animation functionality

### When to Use
The Reveal component provides smooth animation effects for revealing content by adjusting its width or height dimensions. Common use cases include:

- **Progressive content disclosure** - Gradually reveal sections of content as users interact with navigation or scroll through pages
- **Collapsible interface elements** - Animate the opening and closing of panels, sidebars, or content sections
- **Dynamic UI transitions** - Smoothly reveal or hide content based on user actions, form states, or application logic
- **Onboarding flows** - Create engaging step-by-step reveals for tutorials, feature introductions, or guided tours
- **Search result reveals** - Animate the appearance of search results, filters, or dynamic content loading
- **Responsive layout transitions** - Smooth transitions when showing/hiding content in mobile hamburger menus or responsive designs
- **Modal and dialog animations** - Add entrance and exit animations for overlay content like modals, popovers, or tooltips
- **Card interactions** - Reveal additional content or details within cards or list items on hover or click
- **Form field animations** - Show/hide conditional form fields with smooth reveal transitions
- **Dashboard widget reveals** - Animate the display of charts, metrics, or content widgets as they load or become relevant

### Architecture
**No Coupling**: Reveal is a standalone animation component that operates independently without requiring other components. It provides a complete animation solution with customizable direction, duration, and entrance/exit effects that can be applied to any content.

## Components

### Reveal

**Purpose:** Provides smooth animation effects for revealing content by gradually adjusting its width or height dimensions, with configurable direction, timing, and entrance/exit effects.

**Interface:**
```typescript
interface RevealProps {
  // Animation direction
  direction?: 'vertical' | 'horizontal'; // Direction of reveal animation (default: 'vertical')
  
  // Timing configuration
  duration?: number; // Animation duration in milliseconds (default: 300)
  
  // Animation effects
  enter?: AnimationStyles | boolean; // Entering animation styles or enable/disable (default: true)
  exit?: AnimationStyles | boolean; // Exiting animation styles or enable/disable (default: true)
  
  // Content and lifecycle
  children: React.ReactNode; // Content to be revealed with animation
  onStart?: () => void; // Callback fired when animation starts
  onEnd?: () => void; // Callback fired when animation completes
  
  // Visual control
  appear?: boolean; // Whether to animate on initial mount (default: true)
  
  // Standard component props
  className?: string; // Custom CSS classes for styling
  style?: React.CSSProperties; // Inline styles for the wrapper element
}

interface AnimationStyles {
  opacity?: number; // Opacity value (0-1)
  transform?: string; // CSS transform property
  [key: string]: any; // Additional CSS properties
}
```

## Best Practices

### Usage Guidelines
- **Choose appropriate direction** - Use vertical reveal for expanding content sections and horizontal reveal for sidebars or panel-style interfaces
- **Set reasonable durations** - Keep animations between 200-500ms for optimal user experience; shorter for simple reveals, longer for complex content
- **Combine with opacity effects** - Use enter/exit opacity animations alongside dimension changes for smoother visual transitions
- **Maintain content accessibility** - Ensure revealed content is properly announced to screen readers and respects user motion preferences
- **Consider performance impact** - For content with many nested elements, test animation performance and consider using CSS transforms for better optimization

### Animation Guidelines
- **Match animation to content type** - Use quick reveals (200-300ms) for UI controls and slower reveals (400-500ms) for content-heavy sections
- **Coordinate multiple reveals** - When revealing multiple sections, stagger timing by 100-150ms intervals to create natural flow
- **Provide immediate feedback** - Start animations immediately upon user interaction to maintain responsive feel
- **Use consistent timing** - Establish standard duration values across your application for predictable user experience
- **Consider entrance effects** - Combine reveal animations with subtle transform effects (scale, translate) for more engaging transitions

### Performance Considerations
- **Optimize for mobile devices** - Test reveal animations on lower-powered devices and adjust duration/complexity as needed
- **Limit simultaneous animations** - Avoid running multiple reveal animations simultaneously to prevent performance issues
- **Use CSS transforms wisely** - Prefer transform properties over changing layout properties for better rendering performance
- **Consider content complexity** - Simpler content can use faster animations; complex layouts may need more time for smooth transitions
- **Monitor memory usage** - For frequently toggled content, ensure proper cleanup of animation resources

### Accessibility
- **Respect motion preferences** - Honor `prefers-reduced-motion` CSS media query by disabling or reducing animations when requested
- **Provide keyboard support** - Ensure all interactive elements that trigger reveals are keyboard accessible
- **Use ARIA attributes** - Implement `aria-expanded`, `aria-controls`, and `aria-describedby` for collapsible content relationships
- **Announce content changes** - Use `aria-live` regions to inform screen readers when content is revealed or hidden
- **Maintain focus management** - Preserve logical focus flow when content is revealed, especially for keyboard navigation

### Pattern 1: Multi-Step Reveal Flow
```tsx
import { Reveal } from '@progress/kendo-react-animation';
import { Button } from '@progress/kendo-react-buttons';
import { useState } from 'react';

// Sequential content revelation for onboarding or tutorials
function MultiStepReveal() {
  const [currentStep, setCurrentStep] = useState(0);
  
  const steps = [
    {
      title: 'Welcome',
      content: 'Welcome to our platform! Let\'s get you started with a quick tour.'
    },
    {
      title: 'Features Overview',
      content: 'Our platform offers powerful tools to streamline your workflow and boost productivity.'
    },
    {
      title: 'Getting Started',
      content: 'You can begin by creating your first project or importing existing data.'
    },
    {
      title: 'Ready to Go',
      content: 'You\'re all set! Start exploring and make the most of your new tools.'
    }
  ];

  const nextStep = () => {
    if (currentStep < steps.length - 1) {
      setCurrentStep(currentStep + 1);
    }
  };

  const prevStep = () => {
    if (currentStep > 0) {
      setCurrentStep(currentStep - 1);
    }
  };

  return (
    <div className="multi-step-reveal">
      <div className="step-progress">
        <span>Step {currentStep + 1} of {steps.length}</span>
        <div className="progress-bar">
          <div 
            className="progress-fill"
            style={{ width: `${((currentStep + 1) / steps.length) * 100}%` }}
          />
        </div>
      </div>

      <Reveal 
        direction="vertical" 
        duration={400}
        key={currentStep} // Force re-mount for each step
        enter={{ opacity: 0, transform: 'translateX(20px)' }}
        exit={{ opacity: 0, transform: 'translateX(-20px)' }}
      >
        <div className="step-content">
          <h2>{steps[currentStep].title}</h2>
          <p>{steps[currentStep].content}</p>
        </div>
      </Reveal>

      <div className="step-navigation">
        <Button 
          onClick={prevStep} 
          disabled={currentStep === 0}
        >
          Previous
        </Button>
        
        <Button 
          onClick={nextStep}
          disabled={currentStep === steps.length - 1}
          primary
        >
          {currentStep === steps.length - 1 ? 'Finish' : 'Next'}
        </Button>
      </div>
    </div>
  );
}
```

### Pattern 3: Responsive Panel Reveal

import { Reveal } from '@progress/kendo-react-animation';
import { Button } from '@progress/kendo-react-buttons';
import { useState, useEffect } from 'react';

// Responsive panel that adapts reveal direction based on screen size
function ResponsivePanel() {
  const [isPanelOpen, setIsPanelOpen] = useState(false);
  const [revealDirection, setRevealDirection] = useState('horizontal');

  useEffect(() => {
    const updateDirection = () => {
      // Use horizontal reveal for desktop, vertical for mobile
      setRevealDirection(window.innerWidth >= 768 ? 'horizontal' : 'vertical');
    };

    updateDirection();
    window.addEventListener('resize', updateDirection);
    return () => window.removeEventListener('resize', updateDirection);
  }, []);

  const togglePanel = () => {
    setIsPanelOpen(!isPanelOpen);
  };

  return (
    <div className="responsive-panel-container">
      <header className="main-header">
        <h1>Responsive App</h1>
        <Button onClick={togglePanel} className="panel-toggle">
          {isPanelOpen ? 'Close' : 'Open'} Panel
        </Button>
      </header>

      <div className="content-area">
        <main className="main-content">
          <h2>Main Content</h2>
          <p>This is the primary content area that adapts based on panel state.</p>
          <p>The panel reveals horizontally on desktop and vertically on mobile.</p>
