# Splitter Components

## Overview

### Purpose
Use this component to generate splitter layout functionality

### When to Use
The Splitter component is ideal for creating resizable layout sections that allow users to dynamically control space allocation within applications. Use this component when you need to:

- Divide a page or container into multiple resizable panes for flexible content organization
- Create dashboard layouts with adjustable panels for data visualization and controls
- Build code editor interfaces with resizable sidebar, main content, and tool panels
- Implement master-detail views where users can resize list and detail sections
- Design document readers with adjustable navigation and content panes
- Create multi-column layouts where users can prioritize different content areas
- Build responsive layouts that adapt to different screen sizes with user-controlled proportions
- Provide collapsible panes that can be hidden to maximize other content areas
- Support accessibility with keyboard navigation for layout adjustments
- Enable persistent layout preferences that maintain user-configured proportions

### Architecture
**No Coupling**: This is a standalone component that can be used independently. The Splitter component encapsulates all its layout functionality within a single element, managing pane sizing, resizing interactions, and layout calculations internally.

## Components

### Splitter {Standalone Component}
A comprehensive layout component that divides content into resizable sections with user-controlled proportions and collapsible panes.

**Purpose:** Enables flexible layout management with multiple resizable panes that users can adjust through dragging splitter bars, with support for minimum/maximum sizes, collapsible sections, and both horizontal and vertical orientations.

**Interface:**
```typescript
interface SplitterProps extends KendoReactComponentBaseProps {
  // Pane Configuration
  panes?: SplitterPaneProps[]; // Controlled pane configuration array
  defaultPanes?: SplitterPaneProps[]; // Initial pane configuration for uncontrolled usage
  
  // Layout Configuration
  orientation?: 'vertical' | 'horizontal'; // Layout direction (default: 'horizontal')
  
  // Event Handlers
  onChange?: (event: SplitterOnChangeEvent) => void; // Fired when panes are resized or collapsed
  
  // Standard Props
  className?: string; // Additional CSS classes
  style?: React.CSSProperties; // Inline styles
  id?: string; // Component identifier
  dir?: string; // Text direction ('ltr' | 'rtl')
  children?: React.ReactNode; // Pane content elements
}

// Pane configuration interface
interface SplitterPaneProps extends KendoReactComponentBaseProps {
  // Size Configuration
  size?: string; // Pane size (percentage or pixel values like '50%' or '200px')
  min?: string; // Minimum pane size to prevent over-shrinking
  max?: string; // Maximum pane size to limit expansion
  
  // Interaction Behavior
  resizable?: boolean; // Allow pane resizing through splitter bars (default: true)
  collapsible?: boolean; // Enable pane collapse/expand functionality
  collapsed?: boolean; // Current collapsed state of the pane
  
  // Content Management
  scrollable?: boolean; // Enable scrolling for overflowing content (default: true)
  keepMounted?: boolean; // Maintain DOM presence when collapsed
  
  // Standard Props
  className?: string; // CSS classes for individual pane
  style?: React.CSSProperties; // Inline styles for pane
  id?: string; // Pane identifier
}

// Change event interface
interface SplitterOnChangeEvent {
  newState: SplitterPaneProps[]; // Updated pane configuration after resize/collapse
  isLast: boolean; // Indicates final event in a drag sequence for performance optimization
  nativeEvent: React.MouseEvent | React.KeyboardEvent | any; // Original DOM event
}
```

## Component Relationships

### Data Flow
The Splitter component manages layout proportions and pane states internally. When users drag splitter bars or collapse panes, the component calculates new dimensions and updates its internal state. For controlled usage, the `onChange` event provides the new pane configuration to parent components for external state management.

### Event Handling
- **onChange**: Primary event fired during resize operations and collapse/expand actions, providing the updated pane configuration and performance optimization flags
- **Resize Events**: Triggered continuously during drag operations, with `isLast` flag indicating the final event for performance-sensitive operations
- **Collapse Events**: Fired when panes are collapsed or expanded through splitter bar controls

## Best Practices

### Usage Guidelines
- Set explicit heights on the Splitter container to ensure proper layout rendering and prevent content overflow
- Use percentage-based sizing for responsive layouts and pixel-based sizing for fixed-width sidebars or toolbars
- Implement minimum and maximum size constraints to prevent panes from becoming unusable when resized
- Consider the `keepMounted` prop for panes with expensive content that should remain in the DOM when collapsed
- Use the `isLast` flag in onChange events to optimize performance by deferring expensive operations until resize completion
- Provide intuitive collapse/expand controls and keyboard navigation for better accessibility

### Accessibility
- Ensure splitter bars are keyboard accessible using Tab navigation and arrow keys for resizing
- Provide appropriate ARIA labels for splitter bars to describe their function and current state
- Test with screen readers to verify pane content remains accessible during layout changes
- Maintain sufficient contrast for splitter bar visual indicators and resize handles
- Consider providing alternative navigation methods for users who cannot use drag interactions
- Implement focus management to ensure keyboard users can navigate between panes effectively

### Performance
- Use the `isLast` event flag to defer expensive layout calculations until resize operations complete
- Implement debounced handlers for operations triggered by onChange events to prevent excessive API calls
- Consider virtualizing content in large scrollable panes to improve rendering performance
- Minimize inline styles and prefer CSS classes for better rendering optimization
- Use React.memo for pane content components that don't need frequent re-rendering during resize operations
- Profile layout performance with large datasets and implement lazy loading for off-screen content

function ResponsiveSplitterLayout() {
  const [windowWidth, setWindowWidth] = React.useState(window.innerWidth);
  const [isMobile, setIsMobile] = React.useState(window.innerWidth < 768);
  
  React.useEffect(() => {
    const handleResize = () => {
      const width = window.innerWidth;
      setWindowWidth(width);
      setIsMobile(width < 768);
    };
    
    window.addEventListener('resize', handleResize);
    return () => window.removeEventListener('resize', handleResize);
  }, []);
  
  const [panes, setPanes] = React.useState([
    { size: isMobile ? '100%' : '300px', min: '250px', collapsible: true },

        { size: '0px', collapsible: true, collapsed: true }
      ]);
    } else {
      setPanes([
        { size: '300px', min: '250px', collapsible: true, collapsed: false },
        { size: '70%', collapsed: false }
      ]);
    }
  }, [isMobile]);
  
  if (isMobile) {
    // Mobile: Stack layout instead of splitter
    return (
      <div style={{ height: '100vh' }}>
        <div style={{ height: '60px', backgroundColor: '#f8f9fa', padding: '15px' }}>

  return (

      style={{ height: '100vh' }}

function PersistentSplitterLayout() {
  const STORAGE_KEY = 'splitter-layout-preferences';
  
  const [panes, setPanes] = React.useState(() => {
    // Load saved layout from localStorage
    const saved = localStorage.getItem(STORAGE_KEY);
    if (saved) {
      try {
        return JSON.parse(saved);
      } catch (error) {
        console.warn('Failed to parse saved layout preferences');
      }
    }
    
    // Default layout
    return [
      { size: '250px', min: '200px', max: '400px', collapsible: true },
      { size: '50%' },
      { size: '25%', collapsible: true }
    ];
  });
  
  // Debounced save function to prevent excessive localStorage writes
  const debouncedSave = React.useMemo(() => {
    let timeoutId;
    return (panes) => {
      clearTimeout(timeoutId);
      timeoutId = setTimeout(() => {
        localStorage.setItem(STORAGE_KEY, JSON.stringify(panes));
      }, 500);
    };
  }, []);
  
  const handleLayoutChange = (event) => {
    const newPanes = event.newState;
    setPanes(newPanes);
    
    // Save only on the last event of a drag sequence
    if (event.isLast) {
      debouncedSave(newPanes);
    }
  };
  
  const resetLayout = () => {
    const defaultPanes = [
      { size: '250px', min: '200px', max: '400px', collapsible: true },
      { size: '50%' },
      { size: '25%', collapsible: true }
    ];
    setPanes(defaultPanes);
    localStorage.removeItem(STORAGE_KEY);
  };
  
  return (
    <div style={{ height: '100vh' }}>
      <div style={{ height: '50px', padding: '10px', backgroundColor: '#f8f9fa', borderBottom: '1px solid #dee2e6' }}>
        <h2 style={{ margin: 0, display: 'inline' }}>Layout Preferences Demo</h2>

          onClick={resetLayout}
          style={{ float: 'right', marginTop: '5px' }}
        >
          Reset Layout
        </button>
      </div>
      
      <Splitter
        style={{ height: 'calc(100vh - 50px)' }}
        orientation="horizontal"
        panes={panes}
        onChange={handleLayoutChange}
      >
        <div style={{ padding: '20px', backgroundColor: '#f8f9fa' }}>
          <h3>Left Panel</h3>
          <p>Your layout preferences are automatically saved and restored when you reload the page.</p>
          <div style={{ fontSize: '14px', color: '#6c757d', marginTop: '20px' }}>

          <h3>Main Content</h3>

          <div style={{ marginTop: '30px', padding: '20px', backgroundColor: '#e9ecef', borderRadius: '4px' }}>
            <h4>Current Layout:</h4>
            <pre style={{ fontSize: '12px', overflow: 'auto' }}>
              {JSON.stringify(panes, null, 2)}
            </pre>
          </div>
        </div>
        
        <div style={{ padding: '20px', backgroundColor: '#f1f3f4' }}>
